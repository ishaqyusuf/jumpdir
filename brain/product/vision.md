# Vision

## Purpose
Capture why `termcode` exists and what it should feel like.

## How To Use
- Update when product direction changes.
- Keep this focused on user value.

## Vision
Developers often keep projects across several local folders. `termcode` should make those projects feel like one searchable workspace: set roots once, then run, open, and jump into projects by memorable names.

## Project Story
`termcode` exists because day-to-day programming workflows are shifting away from living inside an editor all the time. As AI tools take on more of the code-writing loop, the local development job becomes more about orchestrating projects from the terminal: starting dev servers, jumping between repos, opening folders when context is needed, and keeping multiple tabs moving.

The friction is not the work itself. It is remembering where every project lives.

`termcode` is a small fix for that. Configure the folders where repos live once, then use simple names to list, open, and run project scripts from one terminal. The intended feel is fast, multi-tab friendly, and low-friction without requiring memorized paths or repeated shell habits.

## Product Principles
- Fast to install.
- Easy to explain.
- Safe around folder names and aliases.
- Predictable when duplicates exist.
